# Contributing to Election Intelligence Assistant (EIA)

First off, thank you for considering contributing to EIA! It's people like you that make it a great tool for the community.

## 🏛️ Code of Conduct
By participating in this project, you agree to maintain a professional and respectful environment.

## 🚀 How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issues tracker.
- Provide a clear, descriptive title.
- Describe the exact steps to reproduce the issue.

### Suggesting Enhancements
- Open a GitHub Issue with the tag `enhancement`.
- Explain why this feature would be useful to the civic intelligence mission.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation (`Services.md`, `README.md`).
4. Ensure the test suite passes (`python validate_backend.py`).
5. Make sure your code follows the UX4G design principles for the frontend.

## 🛠️ Development Setup
See the [README.md](README.md) for detailed setup instructions.

## 🎨 Style Guide
- **Python**: Follow PEP 8.
- **Frontend**: Use Vanilla CSS variables for styling; avoid ad-hoc utility classes unless necessary for layout.
- **Git**: Use descriptive commit messages (e.g., `feat: add local polling station lookup`).
